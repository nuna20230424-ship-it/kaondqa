# 법인카드 지출결의서 자동 작성

법인카드 영수증을 판독해 `법인카드 등 정산서` 양식을 채우고, 영수증 파일명을 명목별로 정리하는 스킬이다. 필요하면 매월 정산 폴더 자동생성까지 설정한다.

## 먼저 사용자에게 받을 것

추측하지 말고 없으면 물어볼 것.

1. **영수증 폴더 경로** — 이미지(.jpg/.png/.heic)·PDF가 든 디렉토리
2. **양식 파일 경로** — `법인카드 등 정산서` 엑셀(.xlsx)
3. **파트 접두사** — 예: `개발QA파트_` (명목은 아래 시트에서 가져온다)
4. **명목 시트 CSV 링크** — 구글시트 "파일>공유>웹에 게시>CSV" 링크. E열 비고가 명목 소스. 없으면 사용자가 지정한 고정 명목 또는 계정과목 분류 규칙으로 대체
5. **대상 월** — 예: 2026년 5월 (영수증에서 실제 사용월을 판독하므로 폴더명과 다를 수 있음)
6. **출력 경로** — 기본 `{양식파일명}_작성본.xlsx`

## 명목 분류 (구글시트 E열 비고)

명목 시트가 있으면 각 영수증의 **명목**(예: 웰메이트 / 업무활동비 / 회식비)을 시트에서 가져온다.

- **CSV 읽기** — `https://docs.google.com/spreadsheets/d/{ID}/export?format=csv` 를 받는다. bash curl 이 막히면 PowerShell `Invoke-WebRequest` 사용. 링크 공개(보기) 시트여야 한다
- **시트 구조** — 월별 구역(`5월` 등 헤더 행)으로 나뉜다. 행별 데이터는 **A=일자, B=가맹점명, C=상세, D=금액, E=비고(명목)**. F~L열은 별도 예산요약·명단 테이블이므로 무시. 대상 월 구역의 헤더 다음부터 `합계`/빈 행 전까지가 데이터
- **매칭(날짜+가맹점)** — 시트 행과 영수증을 사용일자+가맹점명으로 매칭. 같은 날 여러 건이면 금액으로 구분
- **불일치 처리** — 날짜가 ±1일 다르거나 금액이 다르면 가맹점+근사금액으로 동일 추정하되, **영수증 실제값을 채택**하고 양식 비고(S)에 차이를 기록. 매칭 실패 건은 `[확인필요]`로 보고
- 시트에만 있고 영수증이 없는 행은 보고만 한다

## 작업 1 — 영수증 파일명 정리

각 영수증을 Read로 열어 **사용일자**를 추출하고, 시트에서 **명목**을 가져와 파일명을 바꾼다.

- 규칙 = `{파트}{명목}_{YYYYMMDD}{확장자}` (예: `개발QA파트_회식비_20260522.jpg`)
- 명목 시트가 없으면 사용자가 준 고정 명목 사용 (예: `개발QA파트_업무활동비_`)
- 같은 (명목, 날짜) 조합이 여러 건이면 시간순으로 `_2`, `_3`
- 날짜 판독 실패 파일은 변경하지 말고 목록으로 보고
- **변경 전 미리보기표(원본→새이름, 명목 포함)로 확인을 받고** 적용. 되돌리기용 매핑 로그를 남길 것
- rename 은 임시명 경유(2단계)로 순환 충돌을 피한다

## 작업 2 — 지출결의서 양식 작성

### 양식 구조 (`법인카드 등 정산서`)
- 헤더 5행 / 데이터 6~26행(최대 21건) / 합계 27행에 `=SUM(…6:…26)` 수식 존재(보존)
- B열="법인카드", C열=카드번호 뒤4자리가 미리 입력돼 있으면 덮어쓰지 말 것

### 컬럼 매핑
| 열 | 헤더 | 채울 값 |
|---|---|---|
| D | 사용(승인)일 | 영수증 사용일자 (datetime.date) |
| E | 가맹점 | 가맹점명 |
| F | 공급가 | 공급가액 (정수) |
| G | 부가세 | 부가세 (정수) |
| H | 사용(승인)액(KRW) | 합계 (정수, = F+G) |
| I | 첨부 증빙#(파일명) | 정리된 영수증 파일명 |
| J | 사용 목적 | 적요 `{가맹점} - {용도}` |
| K | 비용 계정 | **명목(시트 E열) — 웰메이트/업무활동비/회식비**. 시트 없으면 계정과목 분류 규칙 |
| L~R | 참석인원·참석자·접대대상·김영란법·인당접대액 | 접대비일 때만 |
| S | 비고 | `[확인필요]`·시트 불일치 등 플래그 |

### 금액 판독 규칙
- 부가세 별도 표기가 없으면 합계의 1/11을 부가세로 계산(일반과세자)
- 마트 등 혼합과세는 공급가 = 합계 − 표기 부가세
- 면세 품목은 부가세 0

### 계정과목 분류 (명목 시트가 없을 때의 대체 규칙)
- 식당·카페·마트(식품) → 복리후생비 / 다이소·문구 → 사무용품비 / 택시·교통 → 여비교통비 / 주유 → 차량유지비
- 모호하면 추측하지 말고 비고(S)에 `[확인필요]` 표기 후 보고

### 기입 시 주의
- openpyxl 사용. 날짜는 진짜 날짜값, 금액은 정수
- 기존 서식·병합셀·합계 수식·선입력값(B/C)을 망가뜨리지 말 것
- 양식 파일이 Excel에서 열려 있으면 저장이 PermissionError 로 막힌다. 그때는 `_갱신본.xlsx`로 저장하고 사용자에게 Excel 종료를 요청
- 21건 초과 시 사용자에게 알리고 협의

## 작업 3 — 검증 및 보고

저장 후 보고한다.
- 처리 건수 / 전체 파일 수
- **명목별 건수**(웰메이트/업무활동비/회식비 …)
- 계정과목 또는 명목별 합계, 월 총 지출 (공급가 + 부가세 = 합계)
- 시트 매칭 실패·날짜/금액 불일치 목록, `[확인필요]` 항목
- 출력 파일 경로

## (선택) 작업 4 — 매월 정산 폴더 자동생성

`scripts/` 의 PowerShell 스크립트로 매월 1일 정산 폴더를 자동 생성한다.

- `monthly_folder.ps1` — 실행월 폴더 `YY.MM` + `영수증` 폴더 + 전월 양식 사본 생성(멱등). 매개변수 `-RefDate`, `-BaseDir`, `-Master`
- `install_task.ps1` — Windows 작업 스케줄러에 매월 1일 09:00 실행 등록

설정
1. `BaseDir`·`Master`·양식 접두사를 환경에 맞게 수정
2. 마스터(빈) 양식을 `Master` 경로에 둔다. 기본양식 파일명은 월을 비워둔 형태(`…_26년 월_…`)가 정상이며 사본 생성 때만 월을 채운다
3. `install_task.ps1` 실행으로 등록

주의
- PowerShell 5.1은 `.ps1`을 **UTF-8 BOM**으로 저장해야 한글이 안 깨진다(BOM 없으면 파싱 오류)
- 정산 폴더명(YY.MM)은 처리월이고 실제 사용일은 영수증에서 판독한다(전월일 수 있음)
