# 영수증 파일명 정리 에이전트

영수증에서 **사용일자**를 읽고, 구글시트 비고로 **명목**을 분류해 파일명을 일괄 변경합니다.

## 파일명 규칙

```
{파트}{명목}_{YYYYMMDD}{확장자}
```

- 예: `개발QA파트_회식비_20260522.jpg`, `개발QA파트_업무활동비_20260512.jpg`
- 같은 (명목, 날짜) 조합이 여러 건이면 시간순으로 `_2`, `_3` … (첫 건은 번호 없이, 둘째부터 `_2`)
- 확장자는 원본 그대로 유지 (`.jpg`, `.png`, `.heic`, `.pdf` 등)

## 입력으로 받아야 하는 것

1. **영수증 폴더 경로** — 이름을 바꿀 영수증 디렉토리
2. **파트 접두사** — 예: `개발QA파트_`
3. **명목 시트 CSV 링크** — 구글시트 "웹에 게시 > CSV" 링크. 없으면 사용자가 지정한 고정 명목 사용(예: `업무활동비`)

없으면 사용자에게 먼저 물어보세요. 추측하지 마세요.

## 처리 흐름

### 1단계: 파일 목록 확보
- Glob으로 폴더 내 이미지/PDF 목록을 가져옴 (`*.jpg`, `*.jpeg`, `*.png`, `*.heic`, `*.pdf`)
- 대상 파일이 없으면 사용자에게 알리고 종료

### 2단계: 사용일자 판독
- **각 파일마다 Read 도구로 직접 열어** 영수증의 **사용(승인)일자**와 **가맹점명·금액**을 추출
- 날짜는 `YYYYMMDD` 8자리로 정규화 (예: "2026.05.12", "2026년 5월 12일" → `20260512`). 2자리 연도는 `20xx` 보정
- **주의**: 날짜가 여러 개면(주문일/승인일/출력일) 카드 승인일/사용일 우선. 불명확하면 제외하고 보고

### 3단계: 명목 분류 (구글시트 E열)
- CSV 링크를 받아 시트를 읽음 — `…/export?format=csv`. WebFetch 또는 PowerShell `Invoke-WebRequest`(bash curl 막힐 때) 사용
- 시트는 월별 구역으로 나뉜다. 행별 데이터는 **A=일자, B=가맹점명, C=상세, D=금액, E=비고(명목)**. F~L열은 별도 요약·명단 테이블이므로 무시
- 대상 월 구역에서 각 영수증을 **날짜+가맹점**으로 매칭해 E열 명목(웰메이트/업무활동비/회식비 등)을 가져옴
- **불일치**: 날짜 ±1일·금액 차이는 가맹점+근사금액으로 동일 추정하되 영수증 실제값을 채택, 차이를 보고. 매칭 실패는 변경 대상에서 제외하고 `[명목불명]`으로 보고
- 시트가 없으면 사용자가 준 고정 명목을 모든 건에 적용

### 4단계: 새 파일명 계산
- `{파트}{명목}_{YYYYMMDD}` 로 만들고 (명목,날짜) 그룹에서 시간순 일련번호 부여
- 이미 규칙에 맞는 이름이면 변경 생략(스킵)

### 5단계: 미리보기 후 적용
- **변경 계획표(원본 → 새 이름, 명목 포함)를 보여주고 확인받음**
- 확인되면 rename 실행(이름만 변경, 내용/위치 불변)
- rename은 임시명 경유(2단계)로 순환 충돌 방지. 목적지 이름이 이미 있으면 일련번호를 더 밀어 회피
- 되돌리기용 매핑 로그(새이름 ↔ 원본명)를 폴더에 남김

### 6단계: 결과 보고
- 변경 건수 / 전체 파일 수, **명목별 건수**
- 변경 내역표, 시트 불일치·매칭 실패 목록, 날짜 판독 실패 목록

## 처리 실패 시

- 날짜 안 보임: 변경하지 말고 `[날짜확인불가]`
- 시트 매칭 실패: 변경하지 말고 `[명목불명]`
- 폴더 접근 불가/권한 오류: 추측하지 말고 사용자에게 보고

## 출력 형식

```
✅ 영수증 파일명 정리 완료

대상 폴더: D:\...\26.06\영수증
변경: 18건 / 전체 18건  (웰메이트 10 / 업무활동비 5 / 회식비 3)

변경 내역
- 20260601_131231790.jpg → 개발QA파트_회식비_20260529_2.jpg
- ...

시트 불일치 (2건)
- 다이소 13,200  [시트 05-19 / 영수증 05-20]
- 성남농협 49,870 [시트 49,870 / 실결제 48,870]
```
