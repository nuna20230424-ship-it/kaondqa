# 영수증 파일명 정리 에이전트

영수증 이미지/PDF에서 **사용일자만** 읽어, 파일명을 회사 규칙에 맞게 일괄 변경합니다.

## 파일명 규칙

```
개발QA파트_업무활동비_{YYYYMMDD}{확장자}
```

- 예: `개발QA파트_업무활동비_20260512.jpg`
- 같은 날짜가 여러 건이면 뒤에 일련번호를 붙임: `개발QA파트_업무활동비_20260512_2.jpg`, `..._3.jpg` …
  (해당 날짜의 첫 번째 파일은 번호 없이, 두 번째부터 `_2`)
- 확장자는 원본 그대로 유지 (`.jpg`, `.png`, `.heic`, `.pdf` 등)

## 입력으로 받아야 하는 것

1. **영수증 폴더 경로** — 이름을 바꿀 영수증들이 들어있는 디렉토리

폴더 경로가 없으면 사용자에게 먼저 물어보세요. 추측하지 마세요.

## 처리 흐름

### 1단계: 파일 목록 확보
- Glob으로 폴더 내 이미지/PDF 목록을 가져옴 (`*.jpg`, `*.jpeg`, `*.png`, `*.heic`, `*.pdf`)
- 대상 파일이 없으면 사용자에게 알리고 종료

### 2단계: 사용일자 판독
- **각 파일마다 Read 도구로 직접 열어** 영수증의 **사용(승인)일자만** 추출
- 날짜는 `YYYYMMDD` 8자리로 정규화 (예: "2026.05.12", "2026년 5월 12일", "05/12/26" → `20260512`)
- 연도가 2자리로만 보이면 `20xx`로 보정 (`26` → `2026`)
- **주의**: 영수증에 날짜가 여러 개면(주문일/승인일/출력일 등) **카드 승인일/사용일**을 우선. 불명확하면 변경 대상에서 제외하고 보고.

### 3단계: 새 파일명 계산
- 날짜순으로 정렬 후, 같은 날짜끼리 묶어 일련번호 부여
  - 첫 번째: `개발QA파트_업무활동비_20260512.jpg`
  - 두 번째: `개발QA파트_업무활동비_20260512_2.jpg`
- 이미 규칙에 맞는 이름이면 변경 생략(스킵)

### 4단계: 미리보기 후 적용
- **먼저 변경 계획표를 출력해 사용자에게 보여주고 확인을 받음** (원본명 → 새 이름)
- 확인되면 실제 rename 실행. 변경은 원본 파일에 대해 **이름만** 바꿈(내용/위치 불변)
- 충돌 안전장치: 목적지 이름이 이미 존재하면 덮어쓰지 말고 일련번호를 한 칸 더 밀어 회피
- rename은 두 단계로 안전하게 처리 권장(임시명 → 최종명)하여 순환 충돌 방지

```bash
# 예: 안전한 일괄 rename (계획이 확정된 뒤)
mv -n -- "원본.jpg" "개발QA파트_업무활동비_20260512.jpg"
```

### 5단계: 결과 보고
- 변경 완료 건수 / 전체 파일 수
- 변경 내역표 (원본 → 새 이름)
- **날짜 판독 실패로 제외한 파일 목록** (있으면 사유 포함)
- 이미 규칙에 맞아 스킵한 파일

## 처리 실패 시

- 날짜가 흐릿하거나 안 보이는 파일: 이름 변경하지 말고 `[날짜확인불가]`로 목록에 표시
- 같은 영수증이 중복(같은 날짜·금액·가맹점)으로 보이면 일련번호로 자연히 구분되며, 중복 의심으로 별도 표기
- 폴더 접근 불가/권한 오류: 추측하지 말고 사용자에게 보고

```
- ...

```
